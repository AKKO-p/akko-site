Why AKKO Pourquoi AKKO

Data sovereignty and technological
independence are not optional.

Souveraineté des données et indépendance
technologique ne sont pas optionnelles.

Every year, organizations spend hundreds of thousands on proprietary analytics licenses — while their data, their models, and their processes become increasingly locked inside closed ecosystems. AKKO reverses this equation with an open-source, Kubernetes-native analytics platform you fully own.

Chaque année, les organisations dépensent des centaines de milliers d'euros en licences propriétaires — pendant que leurs données, leurs modèles et leurs processus s'enferment toujours plus dans des écosystèmes fermés. AKKO inverse cette équation avec une plateforme analytique open source, Kubernetes-native, que vous possédez entièrement.

Data Sovereignty

Your data never leaves your infrastructure. Apache Iceberg open table format, Apache Polaris catalog, MinIO object storage. No proprietary format. No exit barrier. Total portability.

Cost Control

Permissive open-source engines under the hood. Deploy on your existing hardware, any cloud VM, or on-premise servers. Pay for compute and — if you choose AKKO commercial support — for platform engineering, not for proprietary lock-in.

AI-Native Architecture

LLMs running locally via Ollama (Qwen 2.5-Coder 7B), unified API gateway via LiteLLM, MCP protocol for tool calling, RAG with pgvector. Zero API calls to external providers. Your AI, your models, your rules.

Composable by Design

Every layer is replaceable without touching the others. Swap the SQL engine, the dashboards, the LLM. The platform adapts to you — not the reverse.

Souveraineté des données

Vos données ne quittent jamais votre infrastructure. Format ouvert Apache Iceberg, catalogue Apache Polaris, stockage objet MinIO. Aucun format propriétaire. Aucune barrière de sortie. Portabilité totale.

Maîtrise des coûts

Zéro coût de licence. Construit entièrement avec des composants open source. Déployez sur votre matériel existant, n'importe quelle VM cloud ou vos serveurs on-premise. Vous ne payez que le calcul, jamais le logiciel.

Architecture IA-native

LLM tournant en local via Ollama (Qwen 2.5-Coder 7B), passerelle API unifiée LiteLLM, protocole MCP pour l'appel d'outils, RAG avec pgvector. Zéro appel API vers l'extérieur. Votre IA, vos modèles, vos règles.

Open Lakehouse

Apache Iceberg tables with ACID transactions, time-travel, and schema evolution. Apache Polaris REST catalog with RBAC. MinIO object storage. Your data stays in open formats. Always.

Federated Compute

Spark Connect for distributed compute. Trino for federated SQL across Iceberg and PostgreSQL. Airflow for orchestration with automatic OpenLineage tracking. State-of-the-art engines.

Sovereign AI

Ollama runs LLMs locally (Qwen 2.5-Coder 7B, Qwen 2.5 3B). LiteLLM provides an OpenAI-compatible gateway. MCP servers enable tool calling. pgvector for RAG embeddings. jupyter-ai for conversational AI in notebooks. Zero external API calls.

Automated Governance

OpenMetadata for catalog, lineage, PII classification, glossary, and data quality tests. Domains, data products, ownership. OpenLineage for automatic pipeline lineage. Governed from day one.

Enterprise Security

SSO wired across the whole platform. Fine-grained authorisation as policy-as-code — row-level filtering, column masking, RBAC propagated to every service. Authenticate once, access everything.

Full Observability

Prometheus metrics. Grafana dashboards (4 pre-built). Loki for centralized logs. AlertManager for alerts. Every service monitored, pre-configured, and ready.

Full IDE in the Browser

code-server (VS Code) with 20 pre-installed extensions. JupyterHub with Python, R, and Julia kernels. Quarto for publishing reports. No local install required. Open a tab, start building.

Self-Provisioning Dashboards

Apache Superset with auto-provisioned datasets, charts, and dashboards. SQL Lab for ad-hoc queries. Connected to Iceberg and PostgreSQL through Trino federation.

ML Experiment Tracking

MLflow tracks experiments, registers models, and manages the full ML lifecycle. Integrated with JupyterHub notebooks, MinIO for artifact storage, and PostgreSQL for metadata. From prototype to production.

AI-Native Cockpit

A control plane with built-in AI chat. Query your data catalog, check service health, interact with your data — all from a unified interface. Powered by LiteLLM and MCP servers.

Centralized Audit Trail

Events from every layer (auth, authorisation, queries, pipelines, storage access) centralised into a single dashboard. Who accessed what, when, and why — regulatory compliance out of the box.

dbt Semantic Layer

AI SQL Functions

About À propos

Why we build AKKO.

Pourquoi nous construisons AKKO.

Regulated organisations in Europe are stuck between two bad options: cloud lakehouses whose terms, pricing and data residency they don't control, or years of in-house integration stitching the same open-source components together again and again.

AKKO is the integration layer. We pick the best-in-class open-source component for each part of a modern data platform, wire the SSO through every tool, propagate the governance once, harden it for production and ship it as one Helm chart you own. The value isn't the components — they're free and public. The value is the ten-thousand-hour assembly you no longer do yourself.

Compliance is always a property of the deployment, not the software. AKKO ships the technical controls a regulated team needs to pass an audit — fine-grained RBAC, data lineage, PII classification, full audit trails, air-gapped deployment. Your auditor still signs the paper; we make sure the platform doesn't stand in the way.

We work with three client tiers: self-hosted Community, Enterprise support (signed SLA + hardened images + attestations), and Managed SaaS on sovereign European infrastructure. Pricing is scoped per deployment. Pilots between €0 and €5k against a signed case study.

Contact: hello@akko-ai.com. The team is based in France. AKKO is an Apache-ecosystem citizen — we ship improvements upstream when we fix them.

Questions fréquentes
sur AKKO.

How is AKKO licensed and priced?

AKKO is built on open-source components (Apache 2.0, MIT, BSD). The core platform is available for self-hosted evaluation and community use. Enterprise engagements — 24×7 support, SLA, hardening, compliance attestations, managed SaaS — are commercial offerings. Talk to us to scope a pilot or a production contract.

How does AKKO compare to proprietary analytics platforms?

AKKO provides equivalent capabilities — open lakehouse (Apache Iceberg), federated SQL (Trino), distributed compute (Spark), ML tracking (MLflow), data governance (OpenMetadata), and AI — deployed on infrastructure you control (on-premise, sovereign cloud, or managed by us). No per-query or per-seat billing locked to a vendor console — you keep control of the deployment and the data.

Can AKKO run in air-gapped (disconnected) environments?

Absolutely. AKKO is designed for air-gapped deployment from day one. All container images, AI models, and configurations can be pre-loaded. Zero internet dependency at runtime. This makes AKKO ideal for defense, classified, and high-security environments.

What Kubernetes distributions are supported?

AKKO deploys to any conformant Kubernetes cluster: k3d and k3s for local development, Amazon EKS, Google GKE, Azure AKS, Red Hat OpenShift, and bare-metal Kubernetes. The same Helm chart works everywhere.

How long does it take to deploy AKKO?

About five minutes from git clone to a fully running platform: SSO configured, monitoring active, and AI agents ready. One Helm command handles everything.

Does AKKO include AI capabilities?

Yes. AKKO includes sovereign AI out of the box: local LLMs, an OpenAI-compatible API gateway, MCP servers for sovereign AI agents (Claude, Cursor), AI SQL functions (sentiment, PII, classification, embeddings), and RAG pipelines. All AI processing happens on your infrastructure — zero data sent to external APIs.

What are MCP Servers and how do they work?

MCP (Model Context Protocol) servers let AI agents like Claude and Cursor connect directly to your AKKO platform. They can query Trino via SQL, explore your OpenMetadata catalog, and interact with your data — all on your infrastructure. No data leaves the perimeter. It turns AKKO into an AI-native analytics platform where AI assistants understand your data context.

What is the dbt semantic layer?

AKKO includes dbt Core (Apache 2.0 license) for versioned SQL transformations. Define metrics, data tests, and documentation as code. dbt replaces proprietary semantic layers like Cube.dev or Looker. Orchestrated by Airflow and cataloged by OpenMetadata for full lineage visibility.

Can AKKO detect banking fraud in real time?

Yes. AKKO combines Spark for real-time transaction processing, Apache Iceberg for historical analysis, Trino for federated SQL across all data sources, and MLflow for ML model versioning. OPA policies enforce row-level filtering so each team sees only their authorized data. A complete banking fraud pipeline deploys in minutes.

Does AKKO help meet RGPD and NIS2 obligations?

AKKO ships the technical controls a regulated deployment needs: data stays on your infrastructure (RGPD data residency), Keycloak SSO with 5 RBAC roles, OPA column masking for PII, audit trails via OpenMetadata, encrypted service-to-service communications, Grafana/Prometheus alerting for NIS2 incident reporting. Compliance itself is a property of the deployment plus your internal processes plus the auditor — AKKO gives you the foundation; the certification is earned by your organisation.

Can AKKO handle health data and epidemiological surveillance?

Absolutely. AKKO supports air-gapped deployment for health ministries and hospitals. Spark processes large epidemiological datasets, PostGIS enables geospatial analysis, Superset creates interactive dashboards, and Airflow orchestrates automated pipelines. All processing stays on-premise — critical for patient data sovereignty.
